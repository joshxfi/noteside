Noteside is built to be fast and light first, and to feel instant no matter how big
your notebook gets.

## A native shell

The app is a thin Rust shell ([Tauri](https://v2.tauri.app)) drawing through your
operating system's own webview — no background sync daemon and no separate indexer
process. It launches instantly, stays light on memory, and ships as a small native
binary.

## Everything in memory

Opening a notebook scans it once into a rebuildable in-memory index. There's no
database to build, migrate, or keep in sync — search reads straight from memory, and
your notes stay plain Markdown files on disk.

## The numbers

Measured with `cargo bench` on a modern laptop:

| Operation                      | Result                                               |
| ------------------------------ | ---------------------------------------------------- |
| Fuzzy file & title search      | **sub-millisecond**, even across 50,000 notes        |
| Content search (`:grep`)       | a few milliseconds across thousands of notes         |
| Opening a 10,000-note notebook | ~150 ms, one time                                    |
| Scrolling the sidebar          | virtualized past 100 notes — only visible rows mount |
| Backlinks                      | computed in Rust, off the UI thread                  |

<Callout>No database, no daemon, no indexer to wait on — the fast path is the only path.</Callout>
